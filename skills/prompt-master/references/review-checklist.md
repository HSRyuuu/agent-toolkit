# 프롬프트 리뷰 체크리스트

기존 프롬프트를 리뷰하거나 작성한 프롬프트를 자가 점검할 때 사용한다.

---

## D.1 명확성 검사

| 검사 항목 | 판단 기준 | 개선 방법 |
|----------|----------|----------|
| 모호한 지시 | "적절히", "잘", "가능하면" 등 추상 표현이 있는가 | 수치화된 임계값으로 교체 (C.2) |
| 상충하는 지시 | 프롬프트 내에서 서로 모순되는 규칙이 있는가 | 우선순위를 명시하거나 하나를 제거 |
| 암묵적 가정 | LLM이 알 수 없는 배경지식을 전제하는가 | 필요한 컨텍스트를 명시적으로 제공 |
| 범위 불명확 | "등", "기타" 같은 열린 범위가 있는가 | 구체적 항목을 나열하거나 범위를 한정 |

---

## D.2 구조 검사

| 검사 항목 | 판단 기준 | 개선 방법 |
|----------|----------|----------|
| 강조 남용 | Level 3 이상 강조가 전체의 30%를 초과하는가 | 핵심 1~2개만 고강조, 나머지를 Level 1~2로 하향 |
| 경계 부재 | 역할/단계/출력이 분리 없이 뒤섞여 있는가 | XML 태그 또는 마크다운 헤더로 구조화 |
| 배치 오류 | 핵심 규칙이 프롬프트 중간에 묻혀 있는가 | Primacy/Recency Zone으로 이동 (B.9) |
| 불필요한 반복 | 의도적 반복이 아닌 단순 중복이 있는가 | 중복 제거, 필요 시 형태를 바꾼 반복으로 전환 |

---

## D.3 효과성 검사

| 검사 항목 | 판단 기준 | 개선 방법 |
|----------|----------|----------|
| 예시 부재 | 추상적 규칙만 있고 구체적 예시가 없는가 | Few-shot 예시 추가 (B.2) |
| 검증 부재 | 완료 조건이나 품질 기준이 명시되지 않았는가 | 검증 프로토콜 추가 (B.8) |
| 부정 명령 과다 | NEVER/DO NOT이 ALWAYS보다 많은가 | 긍정 프레이밍으로 전환 (B.7) |
| 탈출 조건 없음 | 반복/재시도 로직에 종료 조건이 없는가 | 서킷 브레이커 또는 최대 반복 횟수 추가 |
| 전략 부재 | 프롬프팅 기법 없이 단순 명령만 나열되어 있는가 | Part A 전략 선택 가이드 참조하여 적절한 기법 적용 |
