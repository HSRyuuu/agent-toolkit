# 비교 리포트 markdown 스키마

`scripts/compare_excel.py`가 생성하는 markdown 리포트 형식. 사용자가 한눈에 변경점을 검토할 수 있도록 고정된 구조를 따른다.

## 전체 구조

```markdown
# 엑셀 변경 비교 리포트

- before: <원본 경로>
- after:  <출력 경로>
- generated_at: 2026-05-11T14:32:11
- protected_sheets: [표지, 개정이력, IF-AI-006, ...]

## 요약

| 항목 | 값 |
|---|---|
| 시트 추가 | 0 |
| 시트 삭제 | 0 |
| 시트 이름 변경 | 0 |
| 행 추가 (총) | 60 |
| 행 삭제 (총) | 60 |
| 변경 셀 (총) | 480 |
| 보호 영역 변경 | **0** |

## ⚠️ 보호 영역 경고

(보호 영역에 변경이 0이 아닐 때만 이 섹션이 등장)

- 시트 `표지` row=2 col=B: "v1.0" → "v1.1"
- ...

→ 의도하지 않은 변경이라면 출력 파일을 폐기하는 것을 권장합니다.

## 시트 단위 변경

### IF-AI-003-002 (신규 시트)

- 행 수: 0 → 28
- 헤더(11행): No. | 필드명 (한글) | ...
- 마스터 시트(`IF-AI-003-001`)와 동일 양식인지: 예

### IF-AI-003-001 (값 변경)

- 변경 셀: 12개
- 샘플:
  | cell | before | after |
  |---|---|---|
  | B2 | AI-003-001 | AI-003-001 |
  | D2 | 컬렉션 생성 | 컬렉션 생성 |
  | E7 | POST /api/collections | POST /api/collections |
  | ... | ... | ... |

### 표지 (변경 없음)

(변경이 0인 시트는 요약 표에만 나오고 본문에는 생략)
```

## 출력 옵션

```bash
python3 scripts/compare_excel.py \
  --before ORIGINAL.xlsx \
  --after  RESULT.xlsx \
  --protected "표지,개정이력,IF-AI-006,IF-AI-009" \
  --sample-cells 5 \
  --output /tmp/excel_diff_report.md
```

| 옵션 | 기본값 | 설명 |
|---|---|---|
| `--protected` | (없음) | 콤마 구분 보호 시트 목록. 이 시트들에 변경이 있으면 ⚠️ 섹션에 별도 표시. |
| `--sample-cells` | 5 | 시트별로 보여줄 변경 셀 샘플 최대 개수. |
| `--output` | stdout | 리포트 저장 경로. 미지정 시 stdout. |
| `--style-check` | false | true면 셀 값뿐 아니라 폰트/테두리/병합도 비교 (느림). |

## 변경 카운트 정의

- **시트 추가/삭제**: before와 after의 시트 이름 집합 차집합으로 계산.
- **시트 이름 변경**: 휴리스틱(시트 인덱스 + 헤더 시그니처)으로 매칭. 매칭 실패 시 추가/삭제로 분류.
- **행 추가/삭제**: 시트별 `max_row` 차이 + ID 컬럼 prefix 매칭으로 계산. ID 컬럼이 없으면 단순 max_row 차이만 보고.
- **변경 셀**: before/after 모두 존재하는 시트의 사용된 모든 셀(`A1:max`)을 비교. 값이 다른 셀의 개수.

## 의사결정 가이드

리포트를 본 LLM/사용자가 다음과 같이 판단하도록 항상 다음 4가지를 명시한다.

1. **보호 영역 변경 = 0인가?** 0이 아니면 출력을 거절하는 게 기본.
2. **시트 추가/삭제 수가 의도와 일치하는가?** 의도가 "60개 추가"였는데 30개만 추가됐다면 spec 추출 실패 가능성.
3. **변경 셀 수가 비정상적으로 큰가?** 의도와 비교해 한 자릿수 이상 차이가 나면 매핑 추론 오류 가능성.
4. **샘플 변경 셀의 before/after가 매핑과 일치하는가?** 한 시트씩 눈으로 확인.
