# 안전 규칙 (위반 금지)

이 규칙들은 모든 실행에서 반드시 지켜야 한다. 한 번이라도 위반하면 사용자의 원본 양식이 망가질 수 있다.

## R1. 입력 xlsx를 절대 덮어쓰지 않는다

- 모든 결과는 새 파일에 저장. 출력 경로는 사용자가 명시하거나, `<원본 stem>_v<bump>_<YYYYMMDD>.xlsx` 형태로 자동 생성.
- 출력 경로가 입력 경로와 같으면 즉시 에러로 중단.
- "옵션으로 in-place"를 허용하지 않는다. 사용자가 원하면 본인이 결과 파일을 원본 위치로 옮기면 된다.

## R2. 보호 영역은 읽지도 쓰지도 않는다

- "보호 영역" = 사용자가 명시한 시트/행 + 자동 추론된 보호 시트(표지, 개정이력 등) + 전략에 무관한 다른 prefix 시트들.
- 갱신 스크립트는 보호 영역 시트 객체를 절대 참조하지 않는다 (`wb[sheet_name]`로 접근 자체 금지).
- 보호 영역 행은 위치가 다른 행 추가·삭제로 자동 밀려도 무방하지만, 셀 값/스타일은 그대로여야 한다. 비교 리포트가 변경을 감지하면 WARNING.

## R3. 스타일은 흉내내지 않고 복제한다

- 신규 시트는 항상 `wb.copy_worksheet(master_sheet)`로 만든다. 빈 시트를 만들고 폰트·테두리·병합·열 너비를 직접 입히는 방식 금지.
- 신규 행은 기존 행의 셀별 `_style` (또는 `font/fill/border/alignment/number_format` 묶음)을 `copy.copy()`로 복제. 직접 객체를 만들어 적용하지 않는다.
- 병합 영역, 열 너비, 행 높이, 데이터 유효성, 조건부 서식, 이미지 — `copy_worksheet`가 기본 처리. 행 단위 복제 시에는 병합/유효성/조건부서식이 자동 따라오지 않으므로, row_append 전략은 헤더 행에 그런 요소가 없는 시트에서만 사용한다.

## R4. 사용자 승인 없이 실행하지 않는다

다음은 모두 사용자에게 명시적으로 보여주고 동의를 받은 뒤에만 진행:

- 추론한 전략 (row_append / sheet_clone / cell_update)
- 보호 영역 화이트리스트
- 셀 ↔ 필드 매핑
- 신규 ID 패턴 (예: `IF-AI-003-{NNN}`)
- 출력 파일 경로

## R5. 비교 리포트 없이 "완료" 보고 금지

- 매 실행 끝에 `scripts/compare_excel.py`로 원본 vs 결과 diff를 만든다.
- 리포트는 사용자에게 그대로 보여준다. "diff가 없다"가 결론이라도 명시적으로 보여준다.
- 보호 영역에 변경이 감지되면 즉시 WARNING 섹션으로 강조하고, 사용자에게 진행 여부를 묻는다 (기본 동작: 출력 파일을 일단 보관하되 "이대로 둘지 삭제할지" 묻는다).

## R6. 데이터 손실 위험 동작은 별도 확인

- 시트 삭제, 행 50개 이상 삭제, 보호 시트 이름 변경 등은 일반 변경과 별도 카운트해서 사용자에게 따로 보여준다.
- "정말 진행할까요?"를 한 번 더 물어 동의를 받은 뒤에만 스크립트를 실행한다.

## R7. 실패 시 출력 파일을 남기지 않는다

- 스크립트 실행 도중 예외가 발생하면, 부분적으로 저장된 출력 파일은 즉시 삭제한다 (또는 처음부터 임시 경로에 쓰고 성공 시에만 최종 경로로 이동).
- 사용자가 "절반만 처리된" 파일을 실수로 사용하지 않도록.

## R8. 입력 파일은 read-only로 연다

- openpyxl `load_workbook(path, read_only=False)`이지만, 절대 `wb.save(input_path)`를 호출하지 않는다.
- 안전을 위해 입력 파일을 메모리(BytesIO)로 한 번 읽어 거기서 워크북을 로드하는 패턴을 권장.

## R9. 자동 후처리 금지

- "이왕 여는 김에" 다른 시트 정렬, 셀 정렬 보정, 폰트 통일 같은 작업을 하지 않는다.
- 사용자가 명시한 변경 외에는 한 글자도 바꾸지 않는다는 원칙.

## R10. 스크립트 코드는 일회성으로 격리

- 생성한 일회성 스크립트는 `/tmp/excel_update_<timestamp>.py`에만 저장한다.
- 플러그인 스킬 스크립트 디렉터리(`skills/excel-doc-updater/scripts/`)에 새 스크립트를 만들지 않는다 — 그곳은 결정론적 공통 도구만 둔다.
