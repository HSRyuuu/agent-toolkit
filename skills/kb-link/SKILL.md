# kb-link — 위키링크 관계 보강

Obsidian 그래프 뷰의 엣지를 늘리기 위해 문서 간 위키링크를 양방향으로 추가한다.

> **Obsidian 스킬 활용**
> - `obsidian-cli` — 볼트 검색, backlink 조회.
> - `obsidian-markdown` — 위키링크 문법 (`[[name]]`, `[[name|alias]]`).

---

## 핵심 원칙

1. **추가만, 수정 없음** — 기존 본문 한 글자도 바꾸지 않는다. 각 대상 문서 끝의 `## 관련` 섹션에 링크를 추가하거나 섹션을 새로 만들 뿐.
2. **양방향 강제** — A → B 링크를 추가하면 B → A도 추가한다. 그래프의 무방향 엣지로 작동시키려는 의도.
3. **일괄 컨펌** — 영향 문서 목록을 한 번에 보여주고 사용자 승인 후 진행.
4. **git에 손대지 않음**.
5. **atomic write** — 모든 쓰기는 `.tmp` → `mv`.

---

## Step 0. 볼트 경로 확인

`~/.config/kb/path`를 읽어 볼트 절대 경로를 얻는다.

```bash
test -f ~/.config/kb/path && cat ~/.config/kb/path
```

### 파일이 없거나 비어있는 경우

다음 안내 메시지를 출력하고 **사용자 응답을 기다린다** (자동 진행 금지):

```
⚠️  KB 볼트 경로가 설정되어 있지 않습니다.

링크 보강할 문서가 없습니다. 먼저 볼트 경로를 설정하세요.

볼트로 사용할 디렉토리의 **절대 경로**를 알려주세요.
  예: /Users/you/workspace/KnowledgeBase

조건:
  - 디렉토리가 미리 존재해야 합니다.
  - 응답하면 ~/.config/kb/path 에 저장되어 모든 kb-* 스킬에서 공유됩니다.

직접 설정하려면 (대신 셸에서):
  mkdir -p ~/.config/kb && echo "/your/vault/path" > ~/.config/kb/path

설정을 원치 않으면 "취소"라고 답하세요.
```

### 사용자 응답 처리

| 응답 | 동작 |
|---|---|
| 절대 경로 (`/`로 시작) | 디렉토리 존재 확인 → 존재하면 저장 후 진행, 없으면 "디렉토리가 존재하지 않습니다: {경로}" 출력 후 재질문 |
| 상대 경로 또는 `~/...` | "절대 경로(`/`로 시작)를 입력해주세요" 출력 후 재질문 |
| "취소" 또는 빈 응답 | 즉시 종료 |

볼트가 새로 설정된 직후라면 보강할 대상 문서가 없을 가능성이 큼. Step 1에서 자연스럽게 "보강할 대상이 없습니다"로 종료된다.

## Step 1. 대상 선정

| 입력 | 모드 |
|---|---|
| `/kb-link {파일경로}` | 명시 모드 — 해당 문서 1개 |
| `/kb-link` (인자 없음) | 최근 모드 — `created` frontmatter가 최근 7일인 문서 전체 |
| `/kb-link --days 30` | 최근 모드 — 윈도우 30일 |
| `/kb-link --folder 03_Resources/RAG` | 폴더 모드 — 해당 폴더 하위 전체 |

대상 문서가 0개이면 다음 메시지로 종료:

> "보강할 대상이 없습니다."

---

## Step 2. 각 대상에 대해 관련 문서 탐색

각 대상 문서 D에 대해:

1. D의 키워드 추출:
   - frontmatter `tags`
   - 제목 (`# `)
   - `## ` 섹션 제목들
   - 본문 고유명사

2. 볼트에서 매칭되는 다른 문서 찾기:
   - tags 교집합 (1개 이상)
   - 제목·섹션 키워드 일치
   - 본문 grep 보조

3. 이미 D와 위키링크로 연결된 문서는 제외 (양방향 모두). **링크 4종 변형 모두 검사:**

   | 형식 | 예 |
   |---|---|
   | 일반 | `[[B-경로]]` |
   | alias | `[[B-경로\|표시텍스트]]` |
   | anchor | `[[B-경로#섹션]]` |
   | embed | `![[B-경로]]` |

   D 본문에서 위 4종 중 어느 하나라도 B를 가리키면 "이미 연결됨"으로 판정. 정규식 예: `!?\[\[B-경로(\|[^\]]*)?(#[^\]]*)?\]\]`.

4. **상위 5개**까지만 후보로 채택.

`raw/`, `00_Inbox/` 폴더는 후보에서 제외.

---

## Step 3. 일괄 제시 + 컨펌

```
=== KB Link 보강 계획 ===

대상: 03_Resources/RAG/rag-단점.md
  + [[03_Resources/RAG/RAG-개념]]          (tags 3개 일치)
  + [[03_Resources/RAG/벡터DB-비교]]        (제목 키워드 일치)
  + [[03_Resources/cache-augmented-LLM]]    (섹션 키워드 일치)

대상: 03_Resources/cache-augmented-LLM.md
  + [[03_Resources/RAG/rag-단점]]           (양방향 강제)

총 영향 문서: 5개 (대상 2 + 추가될 양방향 상대 3)
진행할까요? [Y/n/부분선택]
```

- **승인** → Step 4로 진행.
- **거부** → 종료.
- **부분 선택** → `1,3` 같은 번호 지정. 선택된 쌍에 대해서만 진행.

---

## Step 4. 양방향 링크 추가

각 승인된 쌍 (A, B)에 대해:

### 4-1. A 파일에 B 링크 추가

A의 본문 끝에 `## 관련` 섹션이 있는지 확인.

- **있음**: 섹션 내용을 읽고, `[[B]]`가 없으면 한 줄 추가.
  ```markdown
  - [[B-경로]]
  ```

- **없음**: 파일 끝에 새 섹션 추가.
  ```markdown

  ## 관련  <!-- linked 2026-05-22 -->

  - [[B-경로]]
  ```

### 4-2. B 파일에 A 링크 추가 (양방향)

위와 동일하게 B 파일에도 `[[A]]` 추가.

### 4-3. atomic write

```bash
tmpfile=$(mktemp "{대상}.XXXXXX.tmp")
# tmpfile에 전체 새 내용 작성
mv "$tmpfile" "{대상}"
```

부분 append가 아니라 전체 파일을 새로 쓴 뒤 rename.

### 4-4. qmd 인덱스 리빌드 (best-effort)

모든 쌍 처리 후 1회만 실행. 실패 무시.

```bash
qmd index rebuild 2>/dev/null || true
```

## Step 5. 결과 요약

```
KB Link 완료.
- 처리한 문서: 5개
- 추가된 양방향 엣지: 3쌍 (6개 링크)
- 새로 만든 ## 관련 섹션: 2개

Obsidian 그래프 뷰에서 확인하세요.
```

---

## 하지 않는 일

- ❌ 기존 본문(섹션 본문 포함) 수정.
- ❌ 기존 위키링크 변경·삭제.
- ❌ 파일 생성·삭제·이동.
- ❌ `git` 호출.
- ❌ frontmatter 수정.
- ❌ `## 관련` 외 다른 섹션에 링크 삽입.

---

## 트레이드오프 메모

- **5개 상한**: 노이즈 엣지를 늘리지 않기 위함. 정말 모든 관련 문서가 5개를 넘을 만큼 큰 주제면 사용자가 `/kb-link {경로}`로 명시 호출하면 됨.
- **양방향 강제**: 단방향 링크는 Obsidian backlink로 어차피 검출되지만, 그래프 뷰의 시각적 풍부함을 위해 양쪽에 명시 링크를 둔다.
- **`## 관련` 섹션 컨벤션**: 새 문서를 `/kb-add`가 만들 때도 같은 섹션을 쓰므로 자연스럽게 호환된다.

---

## 예시

```
# 단일 문서 보강
/kb-link 03_Resources/RAG/rag-단점.md
  → 3개 후보 발견
  → 일괄 컨펌
  → 양방향 6개 링크 추가

# 최근 1주 추가분 일괄 보강
/kb-link
  → created >= 7일 전 문서 12개
  → 각 문서당 평균 2~3개 링크 제안
  → 일괄 컨펌 후 진행

# 폴더 단위
/kb-link --folder 03_Resources/RAG
  → 해당 폴더 내 모든 문서에 대해 보강
```
