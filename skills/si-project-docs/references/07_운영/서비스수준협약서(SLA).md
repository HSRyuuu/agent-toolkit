# 서비스수준협약서(SLA) 작성 가이드

## 목적
시스템 운영 중 제공해야 할 서비스 품질 기준(가용률, 응답시간, 장애 복구 목표)과 위반 시 처리 기준을 발주처와 수행사 간에 공식적으로 합의하여 운영 책임과 기대 수준을 명확히 한다.

## 작성 시점
전개 단계 완료 직전(오픈 전)에 작성하여 쌍방 서명으로 확정한다. 협약 기간, 목표 수치, 패널티 조건은 계약 협의 시 결정된 내용을 반영한다.

## 메타데이터 헤더

| 항목 | 내용 |
|------|------|
| 문서명 | 서비스수준협약서(SLA) |
| 시스템명 | {시스템명} |
| 버전 | {v0.0} |
| 협약 기간 | {YYYY-MM-DD ~ YYYY-MM-DD ({기간})} |
| 작성일 | {YYYY-MM-DD} |
| 작성자 | {작성자명 (역할)} |
| 발주처 | {발주처 소속/직책} {담당자명} |
| 수행사 | {수행사 소속/직책} {담당자명} |

## 필수 섹션 구조

### 1. 목적
SLA 작성 목적을 1~2문장으로 기술한다.

### 2. 서비스 가용성
연간·월간 목표 가용률, 허용 다운타임, 가용률 산출 방식, 카운트 제외 예외 조건을 표로 정리한다. 가용률 등급(A/B/C/D)별 수치 기준 표를 함께 제공한다.

### 3. 응답시간 기준
서비스 유형별(화면 로딩, API 응답, 특수 기능 응답 등) 응답시간 기준과 측정 방법을 표로 정리한다.

### 4. 장애 복구 목표
장애 등급별 RTO(복구 목표 시간)와 RPO(데이터 복구 목표 시점)를 표로 정리한다. 비상대책방안의 장애 등급 분류와 일치시킨다.

### 5. 유지보수 시간
정기 점검 일정, 긴급 점검 시 사전 공지 기한, 공지 방법, 연간 총 허용 점검 시간을 표로 정리한다.

### 6. 위반 시 처리 기준

#### 6.1 패널티 조건
위반 항목, 조건(임계값), 패널티 내용(감면율, 보고서 제출 등)을 표로 정리한다.

#### 6.2 패널티 면제 조건
패널티가 면제되는 조건(사전 공지 점검, 불가항력, 발주처 요청에 의한 장애, 외부 서비스 장애 등)을 불릿으로 열거한다.

### 7. 모니터링 방법
측정 항목, 측정 도구, 보고 주기를 표로 정리한다.

#### 7.1 월간 SLA 보고서 제출
보고서 제출 일자, 제출 방법, 포함 내용을 불릿으로 명시한다.

### 8. 협약 당사자 서명
발주처, 수행사 각각의 소속/직책, 성명, 서명란, 서명일을 표로 제공한다.

### 변경 이력
버전, 변경일, 변경 내용, 작성자, 승인자를 표로 관리한다.

## 주요 표 템플릿

**서비스 가용성 표**

| 항목 | 기준 |
|------|------|
| 연간 목표 가용률 | **{N}% 이상** |
| 허용 연간 다운타임 | {N}시간 이내 |
| 허용 월간 다운타임 | {N}시간 이내 |
| 가용률 산출 방식 | (전체 시간 - 서비스 중단 시간) / 전체 시간 × 100 |
| 예외 (카운트 제외) | {사전 공지 점검 시간, 불가항력 등} |

**가용률 등급 표**

| 등급 | 가용률 | 월간 다운타임 |
|------|--------|------------|
| A (목표 초과) | {N}% 이상 | {N}분 이내 |
| B (목표 충족) | {N}% ~ {N}% | {N}시간 이내 |
| C (경고) | {N}% ~ {N}% | {N}시간 이내 |
| D (미달, 패널티 적용) | {N}% 미만 | {N}시간 초과 |

**응답시간 기준 표**

| 서비스 유형 | 응답시간 기준 | 측정 방법 |
|-----------|-----------|---------|
| 일반 화면 로딩 | P95 {N}초 이내 | {측정 도구 및 방법} |
| API 응답 | P95 {N}초 이내 | {측정 도구 및 방법} |
| {특수 기능} 응답 (첫 응답) | {N}초 이내 | {측정 도구 및 방법} |
| {특수 기능} 응답 (완료) | {N}초 이내 | {측정 도구 및 방법} |

**패널티 조건 표**

| 위반 항목 | 조건 | 패널티 |
|---------|------|--------|
| 가용률 미달 | 월간 가용률 {N}% 미만 | 해당 월 운영 비용의 {N}% 감면 |
| 가용률 심각 미달 | 월간 가용률 {N}% 미만 | 해당 월 운영 비용의 {N}% 감면 |
| P1 장애 RTO 초과 | {N}시간 초과 | 사례별 협의 |
| 동일 장애 반복 | 동일 원인 월 {N}회 이상 | 원인 분석 보고서 + 재발 방지 계획 제출 |

**협약 당사자 서명 표**

| 구분 | 소속/직책 | 성명 | 서명 | 서명일 |
|------|-----------|------|------|--------|
| 발주처 | {소속} {직책} | {성명} | ____________ | {YYYY-MM-DD} |
| 수행사 | {소속} {직책} | {성명} | ____________ | {YYYY-MM-DD} |

**장애 복구 목표 표**

| 장애 등급 | 정의 | RTO (복구 목표 시간) | RPO (데이터 복구 목표) |
|---------|------|-------------------|-------------------|
| {P1 (긴급)} | {전체 서비스 불가 등 정의} | {N시간 이내} | {N시간 이내} |
| {P2 (심각)} | {핵심 기능 불가 등 정의} | {N시간 이내} | {N시간 이내} |
| {P3 (중요)} | {일부 기능 불가 등 정의} | {N시간 이내} | 해당 없음 |
| {P4 (경미)} | {성능·UI 이슈 등 정의} | {N시간 이내} | 해당 없음 |

**유지보수 시간 표**

| 유형 | 시간대 | 빈도 | 사전 고지 |
|------|--------|------|---------|
| 정기 점검 | {요일 HH:MM ~ HH:MM} | {매월 N회} | {N일 전 이메일·인앱 공지} |
| 긴급 점검 | {발생 시 즉시} | {비정기} | {N시간 전 이메일 공지} |

**모니터링 방법 표**

| 항목 | 측정 도구 | 보고 주기 |
|------|---------|---------|
| 서비스 가용률 | {측정 도구명} | {월간 보고서} |
| 응답시간 | {측정 도구명} | {주간 보고서} |
| 장애 발생 현황 | {측정 도구명} | {장애 발생 시 즉시 + 월간 요약} |
| {추가 측정 항목} | {측정 도구명} | {보고 주기} |

## 작성 팁

- 가용률 목표 수치(99.5%, 99.9% 등)는 시스템의 중요도와 인프라 비용 수준을 고려하여 현실적으로 설정하고, 비상대책방안의 RTO/RPO와 반드시 일치시킨다.
- 패널티 면제 조건에 외부 서비스(LLM API, SaaS 등) 장애를 명시적으로 포함하되, 해당 장애 발생 시에도 사용자 공지와 RTO 내 대응 의무는 유지하는 방식으로 균형을 잡는다.
- 월간 SLA 보고서 제출 규정을 구체적으로 명시(제출 일자, 방법, 포함 항목)하여 운영 기간 동안 정기 보고 체계를 자동화할 수 있도록 한다.
