# 사용자매뉴얼 작성 가이드

## 목적
시스템의 일반 사용자(PM, BA, 개발자 등 현업 담당자)가 주요 기능을 스스로 학습하고 사용할 수 있도록 접속 방법, 화면 구성, 기능별 사용 절차, FAQ, 오류 대처법을 안내한다.

## 작성 시점
전개 단계 완료 직후(공식 오픈 시점)에 초안을 배포하며, 기능 변경이나 UI 개편 시 해당 섹션을 갱신한다.

## 메타데이터 헤더

| 항목 | 내용 |
|------|------|
| 문서명 | 사용자매뉴얼 |
| 시스템명 | {시스템명} |
| 버전 | {v0.0} |
| 작성일 | {YYYY-MM-DD} |
| 작성자 | {작성자명 (역할)} |
| 대상 독자 | {대상 사용자 유형 (예: PM, BA, 개발자)} |

## 필수 섹션 구조

### 1. 시스템 개요

#### 1.1 시스템 소개
시스템의 핵심 목적과 주요 기능 목록을 불릿으로 간략하게 소개한다. 비기술 사용자도 이해할 수 있는 언어로 작성한다.

#### 1.2 시스템 접속 정보
서비스 URL, 지원 브라우저(버전 범위 포함), 최소 해상도, 접속 방법(VPN 필요 여부 등)을 표로 제공한다.

### 2. 로그인 및 화면 구성

#### 2.1 로그인 방법
지원되는 로그인 방식(SSO, 이메일/비밀번호 등)별로 번호 목록으로 단계를 안내한다. 초기 비밀번호 처리 방법 등 주의사항은 블록쿼트(`>`)로 강조한다.

#### 2.2 메인 화면 구성
주요 메뉴와 콘텐츠 영역의 배치를 텍스트 다이어그램(ASCII)으로 표현하여 화면 구조를 직관적으로 전달한다.

### 3. 주요 기능 사용 방법
기능별로 소제목(`###`)을 나누어 각 기능의 사용 절차를 번호 목록으로 작성한다. 실무에서 자주 사용하는 핵심 기능(등록, 생성, 편집·저장, 검토·승인, 다운로드 등) 위주로 구성한다. 유용한 활용 팁은 블록쿼트로 강조한다.

### 4. 자주 묻는 질문 (FAQ)
번호, 질문, 답변 형식의 표로 실제 사용자가 자주 겪는 문제와 해결 방법을 정리한다.

### 5. 오류 메시지 안내
오류 메시지(시스템 표시 문구), 발생 원인, 해결 방법을 표로 정리한다. 사용자가 직접 조치할 수 있는 것과 관리자에게 문의해야 하는 것을 구분하여 안내한다.

### 변경 이력
버전, 변경일, 변경 내용, 작성자를 표로 관리한다.

## 주요 표 템플릿

**시스템 접속 정보 표**

| 항목 | 내용 |
|------|------|
| 서비스 URL | {https://...} |
| 지원 브라우저 | {브라우저명 최신 N버전, ...} |
| 최소 해상도 | {가로 NNNNpx 이상} |
| 접속 방법 | {인터넷 연결 환경 / VPN 필요 여부} |

**FAQ 표**

| 번호 | 질문 | 답변 |
|------|------|------|
| 1 | {자주 묻는 질문} | {구체적인 해결 방법} |
| 2 | {자주 묻는 질문} | {구체적인 해결 방법} |

**오류 메시지 표**

| 오류 메시지 | 원인 | 해결 방법 |
|-----------|------|---------|
| "{오류 메시지 텍스트}" | {발생 원인} | {사용자 조치 방법 또는 문의처} |

## 작성 팁

- 기능 사용 절차는 클릭 대상 UI 요소를 굵게(`**[버튼명]**`) 강조하여 화면에서 바로 찾을 수 있게 작성한다.
- FAQ는 실제 사용자 교육·베타 테스트에서 수집된 질문을 기반으로 작성하면 실용성이 높아진다.
- 오류 메시지는 시스템에서 실제로 출력되는 문구를 그대로 따옴표로 표기하여 사용자가 매뉴얼에서 빠르게 검색할 수 있도록 한다.
