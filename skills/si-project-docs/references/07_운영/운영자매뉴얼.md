# 운영자매뉴얼 작성 가이드

## 목적
IT 운영팀(시스템 관리자)이 시스템을 일상적으로 운영·유지할 수 있도록 아키텍처 구성, 서버 기동·정지 절차, 배치 작업 관리, 모니터링 방법, 백업·복구, 보안 관리, 장애 처리 절차를 제공한다.

## 작성 시점
전개 단계 완료 직후(공식 오픈 시점)에 작성하여 인수인계 교육 시 함께 전달한다. 시스템 변경(인프라 구조 변경, 배치 추가 등) 시 갱신한다.

## 필수 섹션 구조

### 1. 시스템 아키텍처 요약

#### 1.1 운영자 관점의 시스템 구성
프론트엔드, 백엔드 API, 데이터베이스, 캐시, 파일 저장, CDN, 모니터링 등 구성 요소별로 서비스명, 위치(리전/존), 접근 방법을 표로 정리한다. 운영자가 실제로 접근하는 경로 중심으로 기술한다.

### 2. 서버 기동·정지 절차
서비스 기동, 정지(점검 시), 재기동(롤링 재시작) 각각의 실행 명령어를 코드 블록으로 제공한다. 명령어 실행 후 정상 여부를 확인하는 검증 명령어도 함께 포함한다.

### 3. 배치 작업 관리

#### 3.1 배치 작업 목록
배치명, 스케줄(Cron 표현식 또는 트리거 조건), 내용, 로그 위치를 표로 정리한다.

#### 3.2 배치 수동 실행 방법
배치를 긴급하게 수동으로 실행하는 명령어를 코드 블록으로 제공한다.

### 4. 모니터링 방법

#### 4.1 주요 모니터링 지표
지표명, 정상 범위, 경고 임계값, 위험 임계값을 표로 정리한다. LLM/AI API 비용 등 서비스 특화 지표도 포함한다.

#### 4.2 모니터링 도구 접속
도구명, 접속 URL, 접속 조건(VPN 필요 여부, 계정 권한 등)을 표로 제공한다.

#### 4.3 알림 수신 설정
알림 전달 경로(CloudWatch → SNS → 이메일/Slack 등)와 수신 채널을 기술한다.

### 5. 백업 및 복구 절차
DB 백업 목록 확인 명령어와 특정 시점 복구 명령어를 코드 블록으로 제공한다. 상세 복구 절차는 백업및복구계획서를 참조하도록 안내한다.

### 6. 보안 관리

#### 6.1 사용자 계정 관리
퇴사자 처리(시스템 계정 비활성화, 클라우드 IAM 계정 처리) 절차를 명령어 또는 콘솔 경로로 안내한다.

#### 6.2 접근 로그 확인
특정 사용자의 접근 이력이나 비정상 접근을 조회하는 로그 쿼리 예시를 코드 블록으로 제공한다.

### 7. 장애 처리 절차

#### 7.1 빠른 장애 확인 체크리스트
장애 의심 시 운영자가 가장 먼저 실행해야 할 확인 명령어 시퀀스를 코드 블록으로 제공한다(서비스 응답, 서버 상태, DB 상태, 최근 오류 로그 순).

#### 7.2 일반적인 장애 원인 및 1차 조치
증상, 예상 원인, 1차 조치 방법을 표로 정리하여 운영자가 빠르게 판단할 수 있도록 한다.

### 변경 이력
버전, 변경일, 변경 내용, 작성자를 표로 관리한다.

## 작성 팁

- 모든 명령어 예시는 실제 운영 환경의 클러스터명·인스턴스 ID가 포함된 실행 가능한 형태로 작성하되, 비밀번호·토큰 등 민감 정보는 `{변수명}` 형식으로 마스킹한다.
- 배치 작업 목록의 스케줄은 Cron 표현식과 한국어 설명을 병기하여 운영자가 직관적으로 이해할 수 있게 한다.
- 장애 처리 체크리스트는 최상위 계층(서비스 응답 → 서버 상태 → DB 상태 → 로그)부터 순서대로 확인하는 구조로 작성하여 혼동 없이 따라갈 수 있게 한다.
