# 인수인계계획서 작성 가이드

## 목적
프로젝트 완료 후 수행사에서 운영팀(또는 발주처)으로 시스템 운영 책임을 이전하기 위해 인수인계 범위, 일정, 체크리스트, 교육 계획, 서명 절차를 문서화한다.

## 작성 시점
운영 안정화 단계(오픈 후 약 1개월 시점)에 작성을 시작하며, 병행 운영 기간 완료 전에 쌍방 서명으로 확정한다.

## 메타데이터 헤더

| 항목 | 내용 |
|------|------|
| 문서명 | 인수인계계획서 |
| 프로젝트명 | {프로젝트명} |
| 버전 | {v0.0} |
| 작성일 | {YYYY-MM-DD} |
| 작성자 | {작성자명 (PM)} |
| 승인자 | {승인자명 (발주처)}, {승인자명 (운영팀장)} |

## 필수 섹션 구조

### 1. 인수인계 범위
소스 코드, 문서(산출물), 인프라/클라우드 자원, 계정 정보, 데이터, 모니터링, 운영 절차서 등 항목별로 구분, 항목명, 상세 내용을 표로 정리한다.

### 2. 인수인계 일정
단계별(운영자 교육 → 계정 및 인프라 인계 → 병행 운영 → 최종 인수인계 → 하자보수) 기간, 내용, 담당을 표로 작성한다. 인계자→인수자 방향 화살표로 담당을 표현한다.

### 3. 인계 체크리스트
분야별(소스 코드, 인프라 및 계정, 문서, 운영 지식)로 소제목을 나누어 항목, 완료 여부(☐), 비고를 표로 작성한다.

#### 3.1 소스 코드 인계
저장소 접근 권한, README 최신화, 배포 버전 태그, CI/CD 시크릿 이전 등을 체크한다.

#### 3.2 인프라 및 계정 인계
클라우드 IAM 계정, 시크릿 관리자, 외부 API Key, 도메인·SSL 인증서 설정 등을 체크한다.

#### 3.3 문서 인계
전체 산출물 최종본, 운영자 매뉴얼, 사용자 매뉴얼, 장애 대응 절차서, 백업·복구 계획서 전달 여부를 체크한다.

#### 3.4 운영 지식 인계
모니터링 방법, 배포 절차, 장애 대응, 비용 모니터링 교육 완료 여부를 체크한다.

### 4. 교육 계획

#### 4.1 운영자 교육 (IT 운영팀 대상)
날짜, 교육 내용, 강사, 시간을 표로 작성한다.

#### 4.2 사용자 교육 (현업 사용자 대상)
날짜, 교육 내용, 강사, 시간, 대상(역할 및 인원)을 표로 작성한다.

### 5. 서명 및 확인
인계자(수행사 PM), 인수자(운영팀장), 발주처 확인자의 소속/직책, 성명, 서명란, 서명일을 표로 제공한다.

### 변경 이력
버전, 변경일, 변경 내용, 작성자, 승인자를 표로 관리한다.

## 작성 팁

- 인계 체크리스트는 분야별로 분리하여 담당자가 각자의 영역을 독립적으로 확인하고 서명할 수 있게 구성한다.
- 교육 계획은 운영자(IT 담당)와 최종 사용자(현업)를 구분하여 별도 커리큘럼으로 작성한다.
- 하자보수 기간과 SLA 기준을 명시하여 인수인계 완료 후에도 수행사의 지원 범위와 한계를 명확히 한다.
