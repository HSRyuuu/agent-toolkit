# WBS 작성 가이드

## 목적
프로젝트의 전체 작업을 계층적으로 분해하여 단계·과제·작업·세부작업 단위로 일정, 담당자, 진척률을 관리하고, 마일스톤 기준으로 전체 일정을 통제한다.

## 작성 시점
착수/기획 단계에서 초안을 작성하여 발주처 승인을 받는다. 이후 단계 전환 시 또는 일정 변경 발생 시 버전을 올려 갱신한다.

## 메타데이터 헤더

| 항목 | 내용 |
|------|------|
| 문서명 | WBS |
| 프로젝트명 | {프로젝트명} |
| 버전 | {v0.0} |
| 기준일 | {YYYY-MM-DD} |
| 작성자 | {작성자명 (역할)} |
| 승인자 | {승인자명 (발주처)} |

## 필수 섹션 구조

### 1. 작업 분해 구조
WBS ID, 단계(굵게 표시), 과제, 작업, 세부 작업, 담당자, 계획 시작일, 계획 종료일, 완료율을 포함한 통합 표로 작성한다. 단계 행은 전체 기간과 담당 조직을 표시하고, 하위 행은 들여쓰기 없이 WBS ID 계층(`1`, `1.1`, `1.1.1`)으로 구조를 표현한다.

WBS ID 체계 예시:
- `1`, `2`, `3` ... : 단계 (착수/기획, 분석, 설계, 구축, 시험, 전개, 운영 안정화)
- `1.1`, `1.2` ... : 과제
- `1.1.1`, `1.1.2` ... : 작업/세부 작업

### 2. 일정 계획 요약 (간트 차트 개요)
월별 텍스트 간트 차트(ASCII 블록)로 단계별 기간을 시각화한다. 월 단위 헤더와 `████` 블록으로 기간을 표현한다.

### 3. 마일스톤
주요 완료 기준점(마일스톤)별 계획일, 실적일, 상태(✅ 완료 / ⬜ 예정 / ⚠️ 지연)를 표로 관리한다.

### 변경 이력
버전, 변경일, 변경 내용, 작성자, 승인자를 표로 관리한다.

## 작성 팁

- WBS ID는 최대 3계층(`1.1.1`)까지만 표현하는 것이 가독성에 유리하며, 세부 작업이 많은 경우 별도 스프린트 계획서로 분리한다.
- 완료율은 기준일 기준으로 갱신하며, 100% 이외의 중간 값(예: 50%, 80%)은 실제 작업 진행 비율을 반영하고 임의로 기입하지 않는다.
- 간트 차트 텍스트 다이어그램은 월별 헤더와 블록 길이를 맞춰 단계 간 기간 비중을 직관적으로 표현한다.
