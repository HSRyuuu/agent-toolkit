# 위험이슈관리대장 작성 가이드

## 목적
프로젝트 수행 중 발생하거나 예상되는 위험(Risk)과 실제 발생한 이슈(Issue)를 통합 관리하여 프로젝트 일정·품질·비용에 대한 부정적 영향을 최소화한다.

## 작성 시점
착수/기획 단계에서 초기 위험 목록을 등록하고, 프로젝트 전 기간에 걸쳐 지속적으로 갱신한다. 신규 위험·이슈 발생 시 즉시 등록하고, 주간 진행보고서와 함께 상태를 업데이트한다.

## 필수 섹션 구조

### 1. 위험 관리 대장 (Risk Log)
ID(`RISK-NNN`), 유형(위험), 제목, 상세 내용, 영향 분야, 심각도(상/중/하), 발생 가능성(상/중/하), 대응 방안, 담당자, 상태(모니터링/완화됨/종결)를 표로 관리한다.

### 2. 이슈 관리 대장 (Issue Log)
ID(`ISS-NNN`), 유형(이슈), 제목, 상세 내용, 영향 분야, 심각도(상/중/하), 대응 방안, 담당자, 등록일, 조치일, 상태(오픈/진행 중/종결)를 표로 관리한다.

### 3. 현황 요약
위험과 이슈 각각의 전체 건수를 오픈/진행 중/종결로 분류한 집계 표를 제공한다.

### 변경 이력
버전, 변경일, 변경 내용, 작성자를 표로 관리한다.

## 작성 팁

- 위험(Risk)은 아직 발생하지 않은 잠재적 문제이고, 이슈(Issue)는 이미 발생한 문제다. 두 가지를 명확히 구분하여 별도 표로 관리한다.
- 심각도와 발생 가능성의 조합으로 위험 우선순위를 판단하며, "심각도 상 + 발생 가능성 상"인 항목은 즉시 PM이 직접 대응 방안을 수립한다.
- 종결된 위험·이슈는 삭제하지 않고 상태를 "종결"로 변경하여 이력을 유지한다. 프로젝트 종료 보고서 작성 시 참고 자료로 활용한다.
