# 회의록 작성 가이드

## 목적
프로젝트 회의에서 논의된 내용, 결정 사항, 액션 아이템을 공식 기록으로 남겨 이해관계자 간 합의 내용을 추적하고 후속 조치를 관리한다.

## 작성 시점
회의 당일 또는 익일 이내에 작성하여 참석자에게 배포한다. 수정 요청이 있는 경우 D+2일 이내에 반영한다.

## 메타데이터 헤더

| 항목 | 내용 |
|------|------|
| 문서명 | 회의록 |
| 프로젝트명 | {프로젝트명} |
| 회의 번호 | {MTG-NNN} |
| 회의 일시 | {YYYY-MM-DD (요일) HH:MM ~ HH:MM} |
| 회의 장소 | {장소명} + {화상 도구} 화상 병행 / 대면 전용 |
| 작성자 | {작성자명 (역할)} |
| 작성일 | {YYYY-MM-DD} |

## 필수 섹션 구조

### 1. 회의 기본 정보
**회의명:** {회의 목적을 명확히 나타내는 회의명}

**참석자:** 구분(발주처/수행사), 소속, 성명, 참석 방식(대면/화상)을 표로 정리한다. 서기 역할 담당자를 표기한다.

### 2. 안건
회의에서 다룰 안건을 번호 목록으로 사전 정의한다.

### 3. 논의 내용
안건별로 소제목(`###`)을 나누어 작성한다. 보고 내용, 질의응답(Q&A), 의견을 구분하여 기록한다. 주요 발언은 발언자명과 함께 기록하되, 중요한 질의응답은 블록쿼트(`> Q:`, `> A:`) 형식으로 강조한다.

### 4. 결정 사항
회의에서 확정된 사항을 번호, 결정 내용, 관련자로 표로 정리한다.

### 5. 액션 아이템
후속 조치 항목을 번호(`AI-NNN`), 담당자, 내용, 기한으로 표로 정리한다. 회의 번호와 연계된 채번(예: 첫 번째 회의의 첫 번째 AI = AI-001)을 사용한다.

### 6. 다음 회의
다음 회의 일시, 장소, 예고 안건을 표로 기록한다.

문서 하단에 서기와 검토자(PM) 표기를 추가한다.

## 주요 표 템플릿

**참석자 표**

| 구분 | 소속 | 성명 | 참석 방식 |
|------|------|------|---------|
| 발주처 | {소속 및 부서} | {성명} {직책} | {대면 / 화상} |
| 수행사 | {소속} {역할} | {성명} | {대면 (서기) / 화상} |

**결정 사항 표**

| 번호 | 결정 내용 | 관련자 |
|------|---------|--------|
| 1 | {결정된 내용을 명확하게 기술} | {관련 담당자명} |

**액션 아이템 표**

| 번호 | 담당자 | 내용 | 기한 |
|------|--------|------|------|
| AI-{NNN} | {담당자명} | {구체적인 후속 조치 내용} | {YYYY-MM-DD} |

**다음 회의 표**

| 항목 | 내용 |
|------|------|
| 일시 | {YYYY-MM-DD (요일) HH:MM} |
| 장소 | {장소명} + {화상 도구} 병행 |
| 안건 예고 | {다음 회의에서 다룰 주요 안건} |

## 작성 팁

- 회의 번호(`MTG-NNN`)는 프로젝트 전체에서 일련번호로 채번하여 특정 회의를 쉽게 참조할 수 있게 한다.
- 논의 내용의 질의응답은 발언자를 명시하여 책임 소재를 명확히 하고, 발주처 측 주요 발언은 특히 정확하게 기록한다.
- 액션 아이템은 "누가(담당자) + 무엇을(내용) + 언제까지(기한)" 세 가지를 반드시 포함하여 후속 관리가 가능하도록 작성한다.
