# 프로젝트진행보고서 작성 가이드

## 목적
프로젝트의 주간(또는 월간) 진행 현황을 발주처에 정기적으로 보고하여 전체 진척률, 주요 성과, 이슈, 리스크, 차주 예정 작업을 공유하고 투명한 프로젝트 운영을 유지한다.

## 작성 시점
매주 금요일(또는 합의된 주기)에 작성하여 발주처에 배포한다. 보고 번호는 `W-{NN}` 형식으로 프로젝트 시작부터 순차 채번한다.

## 메타데이터 헤더

| 항목 | 내용 |
|------|------|
| 문서명 | 프로젝트진행보고서 |
| 프로젝트명 | {프로젝트명} |
| 보고 번호 | {W-NN ({NN}주차)} |
| 보고 기간 | {YYYY-MM-DD (월) ~ YYYY-MM-DD (금)} |
| 작성일 | {YYYY-MM-DD (금)} |
| 작성자 | {작성자명 (역할)} |
| 보고 대상 | {수신자명 (발주처)} |

## 필수 섹션 구조

### 1. 전체 진행 현황
전체 진척률(계획 대비 실적), 현재 단계, 남은 기간, 결함 현황(Critical/High 및 Medium 이하)을 표로 요약한다. 상태 아이콘(✅ 정상, ⚠️ 주의, ❌ 위험)을 활용하여 한눈에 파악할 수 있게 한다.

### 2. 주간 주요 성과
이번 주에 완료된 주요 작업을 번호, 성과 내용, 담당자로 표로 정리한다. 구체적인 수치(테스트 Pass율, 응답시간 측정값 등)를 포함하여 객관적으로 기술한다.

### 3. 주간 주요 이슈
이번 주에 발생하거나 진행 중인 이슈를 이슈 ID, 이슈 내용, 현재 상태, 조치 계획으로 표로 정리한다. 위험이슈관리대장의 ID(`ISS-NNN`)와 연계하여 참조 가능하게 한다.

### 4. 차주 예정 작업
다음 주에 수행할 작업을 번호, 예정 작업, 담당자, 완료 기준으로 표로 정리한다. 완료 기준은 측정 가능한 형태로 기술한다.

### 5. 리스크 현황
현재 모니터링 중인 위험 항목을 위험 ID, 위험 내용, 현재 상태, 대응 현황으로 표로 정리한다. 위험이슈관리대장의 `RISK-NNN`과 연계한다.

### 6. 비고
특이사항, 발주처에 요청하는 사항, 다음 보고 예정일 등을 자유 형식으로 기술한다.

보고서 하단에 "다음 보고: {YYYY-MM-DD (요일)}"을 명시한다.

### 보고서 현황 (전체 목록)
누적된 보고서 목록을 보고서 번호, 보고 기간, 주요 이슈, 진척률로 표로 관리한다. 현재 보고서는 굵게 표시한다.

## 주요 표 템플릿

| 항목 | 계획 | 실적 | 상태 |

| 전체 진척률 | {N}% | {N}% | {✅ 계획 초과 / ✅ 정상 / ⚠️ 지연} |
| 현재 단계 | {단계명} | {현재 수행 중인 작업} | - |
| 남은 기간 | 약 {N}일 ({목표 일정} 기준) | - | - |
| 결함 현황 (Critical/High) | 0건 | {N}건 | {✅ 정상 / ⚠️ 주의} |
| 결함 현황 (Medium 이하) | {N}건 이내 | {N}건 ({상태}) | {✅ 정상 / ⚠️ 주의} |

| 번호 | 성과 내용 | 담당자 |
|------|---------|--------|

|---------|---------|---------|---------|

| 번호 | 예정 작업 | 담당자 | 완료 기준 |

|---------|---------|---------|---------|
| {RISK-NNN} | {위험 내용 요약} | {완화됨 / 모니터링 / 미조치} | {대응 내용 및 현황 설명} |

**보고서 현황 표**

| 보고서 번호 | 보고 기간 | 주요 이슈 | 진척률 |
|-----------|---------|---------|--------|
| W-{NN} | {YYYY-MM-DD ~ MM-DD} | {해당 주 주요 이슈 또는 성과} | {N}% |
| **W-{NN}** | **{YYYY-MM-DD ~ MM-DD}** | **{현재 보고 주요 이슈}** | **{N}%** |

## 작성 팁

- 진척률은 WBS 기준으로 산정하며, 계획 대비 실적 차이가 ±5%p를 초과하면 사유와 만회 계획을 비고에 반드시 기술한다.
- 이슈와 리스크는 위험이슈관리대장의 ID를 그대로 인용하여 양 문서 간 추적성을 확보한다.
- 차주 예정 작업의 완료 기준은 "완료"가 아닌 "테스트 Pass 10건 완료", "Pass율 95% 달성" 등 측정 가능한 형태로 작성한다.
