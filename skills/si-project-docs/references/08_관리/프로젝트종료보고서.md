# 프로젝트종료보고서 작성 가이드

## 목적
프로젝트 완료 시점에 목표 달성 여부, 일정·비용 실적, 주요 성과, 교훈 및 개선 사항, 미해결 항목을 종합 정리하여 프로젝트를 공식적으로 종료하고 발주처의 최종 승인을 받는다.

## 작성 시점
인수인계 완료 직전(프로젝트 종료 예정일 기준 약 1~2주 전)에 작성하여 발주처와 수행사 책임자의 서명으로 확정한다.

## 메타데이터 헤더

| 항목 | 내용 |
|------|------|
| 문서명 | 프로젝트종료보고서 |
| 프로젝트명 | {프로젝트명} |
| 버전 | {v0.0} |
| 작성일 | {YYYY-MM-DD} |
| 작성자 | {작성자명 (역할)} |
| 승인자 | {승인자명 (발주처)}, {승인자명 (수행사 대표)} |

## 필수 섹션 구조

### 1. 프로젝트 개요
프로젝트명, 계약 번호, 수행 기간, 총 예산, 최종 집행액, 총 투입 인원(M/M), 발주처, 수행사를 표로 정리한다.

### 2. 목표 달성 결과
최초 설정한 목표 항목별로 최초 목표 수치, 달성 결과 수치, 달성률(✅ 초과 달성 / ✅ 달성 / ⚠️ 미달)을 표로 비교한다. 정량 목표(공수 절감률, 오류율, 준수율 등)와 일정 목표를 모두 포함한다.

### 3. 일정·비용 실적

#### 3.1 일정 실적
단계별 계획 기간, 실적 기간, 차이(일수)를 표로 비교한다. 단축 또는 지연된 단계는 굵게 또는 별도 표기로 강조한다.

#### 3.2 비용 실적
비용 구분별(인건비, 인프라, 외부 서비스, 라이선스, 제경비 등) 계획, 실적, 차이를 표로 비교한다. 합계 행에 전체 절감/초과 금액과 비율을 기재한다.

### 4. 주요 성과

#### 4.1 시스템 구축 결과
구축된 시스템의 핵심 사양(기능 범위, 배포 환경, 서비스 URL, 실측 가용성, 성능 지표 등)을 표로 정리한다.

#### 4.2 정량적 효과
운영 기간 실측 데이터를 기반으로 효과 항목별 측정 결과를 표로 제시한다(공수 절감, 초안 승인율, 사용자 만족도 등).

### 5. 교훈 및 개선 사항

#### 5.1 잘된 점 (Keep)
프로젝트에서 효과적이었던 접근 방식, 기술 결정, 프로세스를 번호 목록 표로 정리한다.

#### 5.2 개선 필요 사항 (Improve)
문제가 발생했던 부분과 다음 프로젝트에서 개선할 방안을 번호, 내용, 다음 프로젝트 반영 방안으로 표로 정리한다.

### 6. 미해결 항목 (후속 과제)
프로젝트 종료 시점에 완료되지 않은 항목을 항목, 내용, 후속 담당, 예정 기간으로 표로 정리한다. 하자보수 범위에 해당하는 항목은 명확히 표기한다.

### 7. 공식 완료 서명
프로젝트 공식 완료를 확인하는 서명란을 발주처 책임자, 수행사 책임자, 프로젝트 매니저 3인으로 구성한다.

### 변경 이력
버전, 변경일, 변경 내용, 작성자, 승인자를 표로 관리한다.

## 작성 팁

- 목표 달성 결과 표는 프로젝트 헌장 또는 제안서의 KPI 항목과 1:1로 대응시켜 작성하며, 미달 항목은 원인과 후속 과제에서 보완 계획을 반드시 기술한다.
- 교훈 및 개선 사항은 팀원 회고(Retrospective) 결과를 반영하여 작성하고, "다음 프로젝트 반영 방안"을 구체적인 행동 수준으로 기술하여 조직 지식으로 축적될 수 있게 한다.
- 미해결 항목은 하자보수 계약 범위와 연계하여 후속 담당을 명확히 지정하고, 발주처가 인수 후에도 추적할 수 있도록 ISS ID 또는 별도 티켓 번호를 함께 기재한다.
