> Claude는 **매 세션 시작 시 이 파일을 가장 먼저 읽고**, 현재 작업 영역에 해당하는 lesson을 의식적으로 적용한다.

## 자기계발 순환 (운영 규칙)

이 파일은 단순 회고록이 아니라 **반복 적용을 위한 운영 규칙집**이다. 다음 4단계를 지킨다:

1. **기록 (Record)** — 사용자가 수정·정정·"이렇게 하지 마" / "이거 다시 해" 같은 피드백을 주면, 즉시 아래 **Lessons** 섹션 맨 위에 새 항목을 추가한다. 다음 세션으로 미루지 않는다.
2. **규칙화 (Rule-ify)** — "사용자가 X를 싫어했다"로 끝내지 않는다. **"다음부터 무엇을 다르게 할 것인가"**를 한 문장 규칙으로 적는다. 추상적이면 의미 없다.
3. **반복 적용 (Apply)** — 매 세션 시작 시 lesson 목록을 훑고, 현재 작업과 관련된 항목을 적용한다. 같은 종류의 작업을 시작할 때마다 해당 lesson을 다시 읽는다.
4. **격상 (Escalate)** — 같은 실수가 한 번 더 반복되면 새 항목을 만들지 말고 **기존 항목을 강화**한다 — 규칙을 더 엄격히 다듬거나, 회피 체크리스트를 더하거나, **반복 횟수**를 +1 한다. 반복 횟수가 일정 임계치(권장 3)에 도달하면 `.claude/rules/<area>.md`로 승격할 후보로 표시한다.

> 오류율이 떨어질 때까지 **냉정하게 반복**한다. lesson은 감정적 자기비판이 아니다 — 다음 행동을 바꾸는 도구다.

## Lesson 형식

각 lesson은 정확히 아래 형식으로 적는다. 형식이 다르면 자동 검색·요약이 깨진다.

```markdown
### YYYY-MM-DD — <한 줄 요약>

- **Trigger**: <어떤 작업·요청이었는지 — 1~2줄>
- **문제**: <Claude가 한 행동, 왜 잘못이었는지 — 1~2줄>
- **Rule**: <다음부터 지킬 한 줄 규칙 — 단정적·구체적>
- **적용 영역**: <영향받는 파일·작업 종류·도메인 — 쉼표로 구분>
- **반복 횟수**: 1
- **승격 후보**: no  *(yes로 바뀌면 `.claude/rules/`로 옮길 시점)*
```

### 좋은 Rule 예시 vs 나쁜 Rule 예시

- ❌ "사용자 의도를 더 잘 파악한다" — 너무 추상적
- ✅ "PR 관련 작업 요청 시, `gh pr view`를 먼저 호출하기 전엔 본문을 추측하지 않는다"
- ❌ "테스트를 더 자주 돌린다" — 임계점 없음
- ✅ "백엔드 코드를 수정하면 동일 모듈의 단위 테스트를 변경 즉시 실행한다 (전체 스위트가 아닌 해당 파일)"

## 분기 점검 (선택)

분기 또는 달 단위로 다음을 검토한다:

- **반복 횟수가 3 이상인 lesson** → `.claude/rules/<area>.md`로 승격 후보. 일반 규칙으로 격상하면 이 파일에서는 **요약만 남기고** 상세는 rules 파일로 옮긴다.
- **6개월 이상 미적용된 lesson** → 여전히 유효한지 검토. 유효하지 않으면 "Archived" 표시 후 별도 섹션으로 이동.
- **연관된 lesson 묶음** → 같은 영역의 lesson 3개 이상이 모이면 통합 규칙으로 묶을 후보.

```markdown
## Archived (승격됨 / 무효화됨)

### YYYY-MM-DD — <한 줄 요약> → `.claude/rules/<area>.md`로 승격
(승격 사유 한 줄)
```
